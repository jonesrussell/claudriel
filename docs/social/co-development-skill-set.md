## Facebook

When your AI assistant works in one repo, it has no idea what's happening in the other two. It rebuilds things the framework already provides. It invents patterns the sibling app solved last week. And nobody measures the drift.

I built three Claude Code skills to fix this: one governs daily development against framework conventions, one audits divergence across projects, and one extracts shared code back into the framework. Ran the first baseline audit today. Minoo hit 100% compliance. Claudriel scored 46%, with 5 high-priority divergences and 3 framework extraction candidates identified.

The loop works: develop with guardrails, measure, extract, repeat.

## LinkedIn

When you develop a framework and two applications together, your AI assistant in one repo has zero awareness of the other two. It duplicates framework capabilities and invents divergent patterns.

I wrote three Claude Code skills to close that gap: app development governance (enforces framework conventions via symlinked skills), cross-project audit (measures pattern divergence with structured compliance reports), and framework extraction (moves shared code from apps into the framework).

First baseline audit results: Minoo scored 100% on 13 compliance checks. Claudriel scored 46%, surfacing 5 high-priority fixes and 3 framework extraction candidates. The system caught what manual review would have missed.
