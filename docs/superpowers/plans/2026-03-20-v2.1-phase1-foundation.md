### Task 1: Fix CLI crash on fresh database (#378)

**Problem:** Entity types registered only in `ClaudrielServiceProvider::commands()` (lines 656-670: artifact, operation) aren't registered in domain service providers. The `ensureTable` loop at line 592 calls `getStorage()` for these types, but if the domain provider hasn't registered them yet, `getStorage()` throws, the catch swallows it silently, and the table never gets created. CLI commands then crash with "no such table."

**Root cause confirmed:** `artifact` and `operation` entity types are defined inline in `commands()` (lines 656-670) but are NOT registered via `$this->entityType()` in any domain service provider's `register()` method. The `OperationsServiceProvider` registers `operation` and `integration` but NOT `artifact`.

**Fix:** Register `artifact` in `OperationsServiceProvider::register()` with proper `fieldDefinitions`. Check if any other entity types from `commands()` are also missing from domain providers.

**Files:**
- Modify: `src/Provider/OperationsServiceProvider.php` — add artifact EntityType registration
- Modify: `src/Provider/ClaudrielServiceProvider.php:656-662` — remove duplicate artifact registration from `commands()`
- Create: `tests/Integration/Command/CliOnFreshDatabaseTest.php`

### Task 2: Fix CORS blocking admin SPA in production (#385)

**Problem:** `config/waaseyaa.php` line 45 has `cors_origins` hardcoded to dev-only values: `['http://localhost:3000', 'http://127.0.0.1:3000']`. Production admin SPA at `https://claudriel.ai/admin/` makes `fetch()` calls to `/api/schema/*` and `/graphql`. Waaseyaa CORS middleware rejects requests from origin `https://claudriel.ai` because it's not in the allowed list. PHP-FPM log confirms: `[Waaseyaa] CORS: origin "https://claudriel.ai" not in allowed list`.

**Fix:** Make `cors_origins` configurable via env var. Add production and staging origins.

**Files:**
- Modify: `config/waaseyaa.php:45` — read origins from env, merge with dev defaults
- Create: `tests/Integration/Controller/CorsTest.php`

- [ ] **Step 1: Write the failing test**

```php
// tests/Integration/Controller/CorsTest.php
<?php

declare(strict_types=1);

namespace Claudriel\Tests\Integration\Controller;

use PHPUnit\Framework\TestCase;

final class CorsTest extends TestCase
{
    public function testCorsOriginsIncludesProductionDomain(): void
    {
        // Simulate production env
        $originalEnv = getenv('WAASEYAA_CORS_ORIGINS');
        putenv('WAASEYAA_CORS_ORIGINS=https://claudriel.ai,https://claudriel.northcloud.one');

        // Re-load config
        $config = require dirname(__DIR__, 3) . '/config/waaseyaa.php';

        $this->assertContains('https://claudriel.ai', $config['cors_origins']);
        $this->assertContains('https://claudriel.northcloud.one', $config['cors_origins']);

        // Restore
        if ($originalEnv === false) {
            putenv('WAASEYAA_CORS_ORIGINS');
        } else {
            putenv("WAASEYAA_CORS_ORIGINS=$originalEnv");
        }
    }

    public function testCorsOriginsDefaultsIncludeDevOrigins(): void
    {
        putenv('WAASEYAA_CORS_ORIGINS');

        $config = require dirname(__DIR__, 3) . '/config/waaseyaa.php';

        $this->assertContains('http://localhost:3000', $config['cors_origins']);
        $this->assertContains('http://127.0.0.1:3000', $config['cors_origins']);
    }
}
```

- [ ] **Step 6: Also fix $fetch usage in useCodifiedContext.ts**

`frontend/admin/app/composables/useCodifiedContext.ts` uses Nuxt `$fetch` for API calls (lines 46, 61, 76, 91). Per CLAUDE.md gotcha, `$fetch` resolves against `app.baseURL` (`/admin/`), turning `/api/telescope/...` into `/admin/api/telescope/...` in production.

Replace `$fetch` with native `fetch` in this file:
```typescript
// Before:
const response = await $fetch<{ data: CodifiedContextSession[] }>(
  `/api/telescope/codified-context/sessions?limit=${limit}`,
)

// After:
const response = await fetch(
  `/api/telescope/codified-context/sessions?limit=${limit}`,
).then(r => r.json()) as { data: CodifiedContextSession[] }
```

Apply the same pattern to all 4 `$fetch` calls in this file.
