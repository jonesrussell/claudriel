## Design

### Shared Pattern

All new providers follow the established lazy-singleton pattern used by `CacheServiceProvider` and `TelescopeServiceProvider`:

```php
final class FooServiceProvider extends ServiceProvider
{
    private ?FooInterface $foo = null;

    public function register(): void
    {
        // Framework DI registration when resolver supports set()
    }

    public function getFoo(): FooInterface
    {
        if ($this->foo === null) {
            $this->foo = new ConcreteFoo(/* config */);
        }
        return $this->foo;
    }
}
```

Storage-backed providers use `storage/` directory with SQLite (WAL mode) and graceful fallback to in-memory on mkdir failure.

### #479 Cache: Wire EntityCacheInvalidator

**What:** Connect the existing `EntityCacheInvalidator` (already built in `CacheServiceProvider`) to the Symfony `EventDispatcher` so entity saves trigger tag-based cache invalidation.

**Where:** `ClaudrielServiceProvider` or a boot-phase hook where both `EventDispatcher` and `CacheServiceProvider` are available.

**Acceptance:** Entity save on any cached type (commitment, person, workspace) clears relevant cache bin entries.

### #478 Telescope: Wire Recorders

**What:** Wire `RequestRecorder` as HTTP middleware via `ClaudrielServiceProvider::middleware()`. Wire `CacheRecorder` to cache events if the cache package exposes event hooks.

**Blocked:** `QueryRecorder` and `EventRecorder` require framework hooks that don't exist yet in waaseyaa/database-legacy. Document this gap, don't work around it.

**Acceptance:** HTTP requests appear in telescope storage. Cache operations recorded when cache events are available.

### #480 State: SqlState Provider

**What:** Add `waaseyaa/state` to `composer.json` require (promote from transitive to direct dependency). Create `StateServiceProvider` with `SqlState` backed by `storage/state.sqlite`.

**Interface:** `StateInterface::get(string $key): mixed`, `StateInterface::set(string $key, mixed $value): void`

**Acceptance:** Provider instantiates without error. State round-trips (set then get) in integration test.

### #481 Queue: MessageBusQueue Provider

**What:** Add `waaseyaa/queue` to `composer.json` require. Create `QueueServiceProvider` with synchronous `MessageBusQueue` transport (no async broker yet).

**Interface:** `QueueInterface::dispatch(object $message): void`

**Acceptance:** Provider instantiates. Dispatching a test message invokes the handler synchronously.

### #482 Mail: Mailer Provider

**What:** Add `waaseyaa/mail` to `composer.json` require. Create `MailServiceProvider` with `Mailer` configured from env vars (`MAIL_DSN` or `MAIL_HOST`/`MAIL_PORT`/etc.).

**Interface:** `Mailer::send(Email $email): void`

**Acceptance:** Provider instantiates. In dev, use null/log transport. In production, SMTP transport from env.

Order rationale: start with the greenfield providers to build momentum, then tackle the "finish wiring" issues that touch existing code.

## Testing Strategy

Each provider gets an integration test in `tests/Integration/Provider/`:

- `StateServiceProviderTest` - round-trip set/get
- `QueueServiceProviderTest` - synchronous dispatch
- `MailServiceProviderTest` - instantiation with null transport
- `CacheServiceProviderTest` - entity invalidation after save (extend existing test)
- `TelescopeServiceProviderTest` - request recording after middleware wiring
