## Remaining Issues (12)

### Phase 1: Foundation (2 issues, session 1)

| Issue | Title | Why first |
|-------|-------|-----------|
| #378 | CLI commands crash with 'no such table: artifact' on fresh DB | Blocks all CLI testing |
| #385 | Admin SPA entity listings hang due to CORS blocking schema/graphql | Blocks manual testing |

**#378 root cause:** `artifact` entity type is registered inline in `commands()` but not in any domain service provider's `register()`. The ensureTable loop calls `getStorage('artifact')`, which throws because the type isn't registered yet. The try-catch silently swallows it. Fix: move artifact registration to `OperationsServiceProvider::register()`.

**#385 root cause:** `config/waaseyaa.php` line 45 hardcodes `cors_origins` to dev-only values (`localhost:3000`). Production admin SPA at `https://claudriel.ai` is rejected by Waaseyaa CORS middleware. Also, `useCodifiedContext.ts` uses Nuxt `$fetch` which prepends `/admin/` baseURL in production. Fix: make cors_origins env-configurable, add production origins, fix `$fetch` to native `fetch`.

## Acceptance Criteria

Each issue's AC is defined in its GitHub issue description. A session is done when:
- All issues in the session pass their GitHub-defined acceptance criteria
- Tests pass (existing + new)
- CI is green
