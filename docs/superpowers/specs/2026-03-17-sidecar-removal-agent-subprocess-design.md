# Sidecar Removal & Agent Subprocess Architecture

**Date:** 2026-03-17
**Status:** Approved
**Scope:** Claudriel chat infrastructure

## Problem

Claudriel uses a Docker sidecar (FastAPI + `claude-agent-sdk`) to provide agentic tool access (Gmail, Calendar) in chat. The sidecar was originally a hack to piggyback on Claude Code's MCP tools for Google access before proper OAuth existed.

Now that Google OAuth is implemented via `GoogleTokenManager`, the sidecar is unnecessary infrastructure:

- Extra Docker container to build, deploy, and health-check
- OAuth token mounting hacks (`~/.claude` volume mounts)
- Fallback logic between `SidecarChatClient` and `AnthropicChatClient`
- Two model configs (`CLAUDE_MODEL` vs `ANTHROPIC_MODEL`)
- Session management in Python that duplicates PHP state

## Decision

Remove the sidecar. Keep `claude-agent-sdk`. Run it as a PHP subprocess.

## Architecture

```
Browser <--SSE--> PHP (ChatStreamController) <--JSON-lines/stdout--> Python (agent/main.py) <--HTTP--> PHP API (/api/internal/*)
```

Three layers, each with one job:

| Layer | Responsibility |
|---|---|
| Python | Agentic loop + tool definitions (transport-agnostic, emits JSON-lines) |
| PHP | Business logic, OAuth, tenant isolation, SSE transport |
| Browser | UI rendering |

The Python agent calls back into PHP's internal API for all Google operations. OAuth token management stays in PHP.

No fallback path. If the subprocess fails, PHP emits an error SSE event and logs stderr. The agentic loop is the product — a degraded text-only mode is not equivalent and would rot untested.

## Python Agent Package

```
agent/
  main.py                # Entrypoint: reads request from stdin, runs agentic loop, writes JSON-lines to stdout
  requirements.txt       # claude-agent-sdk, httpx
  tools/
    __init__.py           # Auto-registers all tools
    gmail_list.py         # List inbox messages
    gmail_read.py         # Read a specific message
    gmail_send.py         # Send/reply to email
    calendar_list.py      # List upcoming events
    calendar_create.py    # Create calendar event
  util/
    __init__.py
    http.py               # Thin wrapper for calling PHP internal API (passes account_id, auth token)
```

### Subprocess Contract

**PHP sends to stdin** (single JSON object, then closes stdin):

```json
{
  "messages": [],
  "system": "...",
  "account_id": "...",
  "tenant_id": "...",
  "api_base": "http://localhost:8000",
  "api_token": "short-lived-hmac-token"
}
```

- `messages` — Anthropic-compatible message history
- `system` — system prompt (built by `ChatSystemPromptBuilder`)
- `account_id` — for multi-tenant isolation
- `tenant_id` — for tenant-scoped operations and debugging
- `api_base` — PHP app base URL for tool callbacks
- `api_token` — short-lived HMAC token generated by PHP per-request (see Security Model)

**Python writes to stdout** (JSON-lines, one event per line):

```
{"event": "message", "content": "Let me check your inbox..."}
{"event": "tool_call", "tool": "gmail_list", "args": {"query": "is:unread"}}
{"event": "tool_result", "tool": "gmail_list", "result": {"count": 3, "messages": [...]}}
{"event": "message", "content": "You have 3 unread emails..."}
{"event": "done"}
```

**Error handling:**

- Errors go to stderr (PHP logs them), never stdout
- Exit code 0 = success, non-zero = failure
- Invalid JSON on stdout triggers error SSE event to browser

**PHP maps JSON-lines to SSE:**

```
event: message
data: {"content": "..."}

event: tool_call
data: {"tool": "gmail_list", "args": {...}}

event: done
data: {}
```

## PHP Changes

### New: SubprocessChatClient

Replaces both `SidecarChatClient` and `AnthropicChatClient`:

- Resolves agent path from a configured absolute path (env `AGENT_PATH` or default `{project_root}/agent/main.py`) — never relies on CWD
- Spawns `{agent_venv}/bin/python {agent_path}` via `proc_open()`
- Writes request JSON to stdin, closes stdin
- Reads stdout line-by-line, parses each JSON-line
- Maps events to SSE and flushes to the browser
- Sends SSE keepalive comments (`: ping`) every 15 seconds to prevent proxy/browser timeouts
- Captures stderr for logging
- Enforces a 120-second wall-clock timeout — kills the subprocess via `proc_terminate()` if exceeded
- If the browser disconnects (connection aborted), kills the subprocess immediately
- Handles process exit code (non-zero = error SSE event)

### New: Internal Google API Endpoints

| Endpoint | Tool | Auth |
|---|---|---|
| `GET /api/internal/gmail/list` | `gmail_list` | HMAC token + account_id |
| `GET /api/internal/gmail/read/{id}` | `gmail_read` | HMAC token + account_id |
| `POST /api/internal/gmail/send` | `gmail_send` | HMAC token + account_id |
| `GET /api/internal/calendar/list` | `calendar_list` | HMAC token + account_id |
| `POST /api/internal/calendar/create` | `calendar_create` | HMAC token + account_id |

These are internal-only endpoints. They use a short-lived HMAC token that PHP generates and passes to the subprocess. Each endpoint uses `GoogleTokenManager` to get the user's OAuth token and calls the Google API.

Python never touches Google credentials directly.

### Modified: ChatStreamController

- Remove sidecar availability check
- Remove fallback logic
- Single path: build request -> spawn subprocess -> stream response

### Modified: ChatSystemPromptBuilder

- Remove conditional tool instructions (sidecar-available vs not)
- Tools are always available — simplify prompt accordingly

## Security Model

- **HMAC token format**: `HMAC-SHA256(AGENT_INTERNAL_SECRET, "{account_id}:{timestamp}")`. The token payload is `{account_id}:{timestamp}:{signature}`. PHP generates it per chat request with a 300-second TTL (5 minutes — long enough for a full agentic loop). PHP validates the signature and checks that `timestamp` is within the TTL window on every internal API call. If the subprocess outlives the token, internal API calls return 401 and the tool result propagates the error to Claude.
- **Internal endpoint protection**: `/api/internal/*` routes are protected by HMAC validation only (not firewall-restricted). `AGENT_INTERNAL_SECRET` must be at least 32 bytes of cryptographic randomness. The HMAC is per-request and per-account, so brute-forcing is impractical.
- **Tenant isolation**: PHP enforces tenant boundaries on every internal API call via `account_id` and `tenant_id`.
- **No credential leakage**: Python never sees Google OAuth tokens. It only sees the HMAC token for calling PHP's internal API.
- **No shell access**: Unlike the sidecar, the Python agent has no Bash tool. Tools are limited to the defined Google operations.

- `src/Controller/ChatStreamController.php` — single SubprocessChatClient path
- `src/Domain/Chat/ChatSystemPromptBuilder.php` — remove conditional tool instructions

### Environment changes

- Add `AGENT_INTERNAL_SECRET` — shared secret for HMAC token generation (min 32 bytes)
- `AGENT_PATH` — absolute path to `agent/main.py` (default: `{project_root}/agent/main.py`)
- `AGENT_VENV` — path to Python virtualenv (default: `{project_root}/agent/.venv`)

### Deploy changes

- Create virtualenv: `python3.11 -m venv agent/.venv` (Python 3.11+ required by `claude-agent-sdk`)
- Install deps: `agent/.venv/bin/pip install -r agent/requirements.txt`
- No more Docker sidecar image builds
- No more sidecar deploy tasks
